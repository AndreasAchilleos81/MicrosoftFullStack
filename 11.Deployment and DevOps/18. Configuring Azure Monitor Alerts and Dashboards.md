# Azure Monitor: Alerts & Dashboards Guide

## Overview

This guide covers how to set up **alerts** and **dashboards** in Azure Monitor to track real-time application performance.

---

## Monitoring Your Web App

When viewing a web app in the Azure portal, the **Overview > Monitoring** tab provides basic metrics at a glance:

- HTTP 5xx Errors
- Data In / Data Out
- Response Time
- Request Count

For deeper diagnostics, navigate to the **Monitoring** section of your web app to access richer metrics and log configuration.

---

## Setting Up Alert Rules

Alert rules define the **conditions** under which you get notified about your application's behavior.

### Steps to Create an Alert Rule

1. **Select a Signal** — The trigger that fires the alert (e.g., Response Time).
2. **Configure the Condition** — Choose between:
   - **Dynamic threshold** — Azure Monitor calculates a baseline average and alerts when the metric exceeds it (default sensitivity: Medium).
   - **Static threshold** — You define the exact value (e.g., alert if average response time > 2 seconds).
3. **Set the Check Frequency** — Default: every **1 minute**, with a **5-minute lookback** period.
4. **Create an Action** — Define what happens when the condition is met:
   - Send an email notification
   - Show an alert in the Azure portal
   - Trigger a webhook, Logic App, Azure Function, etc.
5. **Add Details** — Assign resource groups and tags for organization.
6. **Review + Create** — Confirm settings and deploy the rule.

> **Tip:** You can create multiple alert rules for the same metric with different check periods and thresholds to ensure comprehensive coverage.

## Creating a Dashboard

Dashboards give you a **single pane of glass** to view all your metrics and alerts.

### Steps to Create a Dashboard

1. Search for **"Dashboard"** in the Azure portal search box → open **Dashboard Hub**.
2. Click **New** and choose a template:
   - **Blank dashboard** — Start from scratch.
   - **Application Insights dashboard** — Pre-built tiles for app health and performance metrics.
3. Name your dashboard (e.g., `app-metrics`), select your **subscription** and **resource**, then click **Submit**.
4. **Edit** the dashboard to add/remove tiles, resources, or resource groups.
5. Use **filters** to control how individual tiles respond to filter changes.

> **Tip:** A single dashboard can display metrics from **multiple applications**, giving you a unified view across all your web apps.

## Azure CLI Commands (Latest — 2025)

### Authentication

```bash
az login
az account show        # View current subscription
az account set --subscription "<subscription-id>"  # Switch subscription
```

### Metric Alert Rules

```bash
# Create a metric alert rule (e.g., CPU > 90%)
az monitor metrics alert create \
  -n {alertName} \
  -g {resourceGroup} \
  --scopes {resourceID} \
  --condition "avg Percentage CPU > 90" \
  --description "CPU usage alert"

# Create alert with static response time threshold
az monitor metrics alert create \
  -n "ResponseTimeAlert" \
  -g {resourceGroup} \
  --scopes {webAppResourceID} \
  --condition "avg HttpResponseTime > 2" \
  --window-size 5m \
  --evaluation-frequency 1m \
  --description "Alert if avg response time exceeds 2 seconds"

# List all metric alert rules
az monitor metrics alert list --resource-group {resourceGroup}

# Delete a metric alert rule
az monitor metrics alert delete --name {alertName} --resource-group {resourceGroup}
```

### Log Search Alert Rules

```bash
# Create a log search alert rule
az monitor scheduled-query create \
  -g {resourceGroup} \
  -n {alertName} \
  --scopes {resourceID} \
  --condition "count > 10" \
  --description "Log query alert"
```

### Activity Log Alert Rules

```bash
# Create an activity log alert rule
az monitor activity-log alert create \
  -g {resourceGroup} \
  -n {alertName} \
  --scope {subscriptionID} \
  --condition category=Policy
```

### Action Groups

```bash
# Create an action group (e.g., email notification)
az monitor action-group create \
  -g {resourceGroup} \
  -n {actionGroupName} \
  --short-name {shortName} \
  --action email myalias name@example.com
```

### Alert Processing Rules

```bash
# Install required extension
az extension add --name alertsmanagement

# Create an alert processing rule
az monitor alert-processing-rule create \
  --name "AddActionGroupToSubscription" \
  --rule-type AddActionGroups \
  --scopes "/subscriptions/{subscriptionID}" \
  --action-groups "/subscriptions/{subscriptionID}/resourcegroups/{RG}/providers/microsoft.insights/actiongroups/{AG}" \
  --resource-group {resourceGroup} \
  --description "Add action group to all alerts in the subscription"

# List all alert processing rules
az monitor alert-processing-rule list

# Disable an alert processing rule
az monitor alert-processing-rule update \
  --resource-group {resourceGroup} \
  --name {ruleName} \
  --enabled false

# Delete an alert processing rule
az monitor alert-processing-rule delete \
  --resource-group {resourceGroup} \
  --name {ruleName}
```

### Diagnostic Settings

```bash
# List diagnostic settings for a resource
az monitor diagnostic-settings list --resource {resourceID}

# Create diagnostic settings
az monitor diagnostic-settings create \
  --name {diagnosticName} \
  --storage-account {storageAccountID} \
  --resource {resourceID} \
  --logs '[{"category": "AppServiceHTTPLogs", "enabled": true, "retentionPolicy": {"days": 30, "enabled": true}}]'
```

### Metrics

```bash
# List available metric definitions for a resource
az monitor metrics list-definitions --resource {resourceID}

# View metrics for a resource
az monitor metrics list \
  --resource {resourceID} \
  --metric "HttpResponseTime" \
  --aggregation Average
```
